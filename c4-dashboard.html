🏗️ Shard Wallets & Forwarder Contracts
MultiversX splits the network into 3 parallel processing lanes (Shards 0, 1, 2). The DEX pair contract lives on Shard 1. We deploy one forwarder contract and one wallet per shard. 🟢 Green border = same shard as DEX (blindSync works here, fastest). 🟠 Orange = cross-shard (async calls only, tokens need draining). Each worker cycles through all 4 call types via round-robin.
📡 The 4 Call Methods — Must Hit 300+ Each!
MultiversX supports 4 different contract-to-contract call mechanisms. The challenge requires us to use ALL 4. Each type must individually reach 300 successful calls minimum or we lose scoring eligibility. blindSync = instant but same-shard only. The others work cross-shard but are slower and require draining stuck tokens back. Our blaster rotates through all 4 types automatically on every worker.
⚡ blindSync
Fastest method. Same-shard ONLY (Shard 1 forwarder). Tokens return instantly. No drain needed. Our speed weapon.
